# Contributing

Thanks for helping improve PM Delivery Steward.

1. Do not add real customer, employer, meeting, Jira, or personal data.
2. Keep the application local-first and evidence-grounded.
3. Run `npm test` and `npm audit` before opening a pull request.
4. Update `CHANGELOG.md` for user-facing changes.
5. Keep API keys only in `.env`; never include them in code, fixtures, screenshots, or issues.

For larger changes, open an issue first to align on the workflow and privacy impact.
